# /auto — 연속 진행 모드

승인된 체크리스트를 사용자 OK 없이 자율 연속 진행합니다.

## 인자
- `$ARGUMENTS` — 진행할 항목 수 (숫자). 비어있으면 남은 `[ ]` 전체를 진행합니다.

## 전제 조건 (하나라도 불충족 시 즉시 거부)
1. `.plans/`에 활성 플랜이 존재해야 합니다 (`03_checklist.md`에 `[ ]` 항목이 1개 이상).
2. 해당 플랜이 reviewer PASS 상태여야 합니다.

불충족 시 아래를 출력하고 종료:
```
자율 진행 불가: {사유}
→ /task-plan 으로 플랜을 먼저 세우세요.
```

## 시작 전 확인

1. `.plans/`에서 가장 최근 활성 폴더를 찾습니다 (`ls -t` 수정 시간순).
2. `03_checklist.md`를 읽어 남은 `[ ]` 항목 수를 파악합니다.
3. `$ARGUMENTS`가 숫자면 N으로 사용. 비어있으면 N = 남은 전체 항목 수.
4. 시작 보고를 출력합니다:
```
[AUTO] 연속 진행 시작
작업: {플랜 폴더명}
대상: {N}개 항목 ({첫 항목} ~ {마지막 대상 항목})
━━━━━━━━━━━━━━━━━━
```

## 루프 흐름 (각 항목마다 반복)

### 1. 항목 파악 + 에이전트 위임
- `03_checklist.md`에서 첫 번째 `[ ]` 항목을 읽습니다.
- 항목 성격에 따라 적절한 에이전트를 위임합니다:
  - 코드 변경 → developer
  - 문서 수정 → doc-writer
  - 기타 → 항목 내용에 맞는 에이전트 판단

### 2. tester 호출 (코드 변경 시)
- CLAUDE.md "코드 변경 판단 기준" 분류표를 따릅니다.
- 코드/시스템 동작 영향 파일 변경 → tester 호출.
- 문서만 수정 → tester 스킵.

### 3. reviewer 호출 (게이트 패턴)
- reviewer를 호출합니다 (코드 리뷰 모드).
- **PASS** → 4단계로 진행.
- **REVISE** → 자동 수정 후 reviewer 재호출. 최대 3회까지 반복.
  - 3회 초과 시 → **중단** (중단 처리 섹션으로 이동).

### 4. 체크리스트 마킹 + 상태 갱신
- `03_checklist.md`에서 해당 항목을 `[x]`로 마킹합니다.
- `02_context.md` "태스크 진행 상태" 섹션을 갱신합니다:
  - 완료된 항목 추가
  - 다음 항목 기록

### 5. 커밋 생성
- version-control skill 참조하여 항목별 분리 커밋을 생성합니다.
- **push는 하지 않습니다.**

### 6. 진행 로그 출력
```
[AUTO] {완료 순번}/{N} 완료: {항목 제목}
  수정: {파일 목록}
  커밋: {커밋 해시 7자리}
```

### 7. 다음 항목으로
- 완료 수 < N이면 1단계로 돌아갑니다.
- 완료 수 = N이면 완료 처리로 이동합니다.

## 중단 조건
아래 중 하나라도 발생하면 즉시 루프를 멈춥니다:
- reviewer REVISE가 같은 항목에서 3회 초과
- 에이전트 실행 중 에러/블로커 발생
- 사용자가 메시지를 보내면 즉시 자율 모드 해제
- N개 항목 모두 완료

## 중단 시 처리
1. `02_context.md`에 중단 지점을 기록합니다:
```markdown
## 자율 진행 중단
- 중단 항목: {항목 제목}
- 중단 사유: {REVISE 초과 / 에러 내용 / 사용자 개입}
- 완료: {완료 수}/{N}
- 마지막 커밋: {해시}
```
2. 사용자에게 중단 보고:
```
[AUTO] 중단됨 ({완료 수}/{N} 완료)
중단 사유: {사유}
중단 항목: {항목 제목}
→ /restore 로 이어서 진행 가능
```

## 완료 시 처리

### 1. recommender 자동 반영
recommender를 호출한 뒤, 각 제안을 자동 분류하여 처리합니다:

**반영 전 준비:** `03_checklist.md`에 `- [ ] recommender 제안 반영` 임시 항목을 추가합니다 (plan-gate 우회용).

**제안 분류:**
- **단순 수정** (5줄 이내, 단일 파일) → 바로 반영 + 커밋
- **복잡한 수정** (5줄 초과, 다수 파일, 판단 필요) → `/issue`로 GitHub Issue 자동 등록
- **plan-gate 차단 시** → `/issue`로 GitHub Issue 자동 등록

반영 완료 후 임시 항목을 `[x]`로 마킹합니다.

### 2. 전체 요약 보고
```
[AUTO] 연속 진행 완료
━━━━━━━━━━━━━━━━━━
완료: {완료 수}개 항목
커밋: {커밋 해시 목록, 각 1줄}
실패: {있으면 항목명, 없으면 "없음"}
recommender: {반영 N건, 이슈 등록 N건}
━━━━━━━━━━━━━━━━━━
push 할까요?
```

## 주의사항
- 이 모드는 CLAUDE.md "태스크 완료 워크플로우"의 모든 규칙을 따릅니다. 주요 차이는 **"사용자 OK 대기" 단계 스킵**과 **recommender 호출 시점 변경**(항목별 → 전체 완료 후 1회)입니다.
- reviewer, tester 등 게이트 패턴은 절대 스킵하지 않습니다.
- 사용자가 메시지를 보내는 즉시 자율 모드를 해제하고, 현재 항목 완료 후 멈춥니다.
- recommender는 전체 완료 후 1회 호출하고, 제안을 자동 분류하여 반영(단순) 또는 이슈 등록(복잡)합니다.
