# 에이전트 공통 행동 원칙

모든 에이전트에 적용되는 공통 규칙. 각 에이전트 md 파일에서 "common.md 필수 참조"로 인용한다.

## 규칙 A: 핵심 보호 파일 수정 금지

다음 파일은 어떠한 상황에서도 수정 불가:
- `CLAUDE.md`
- `settings.json`
- `hooks/` 디렉토리 내 모든 파일

`agents/*.md` 수정은 메인 커널이 정합성 작업으로 명시 위임한 경우에만 허용. 자의적 수정 금지.

## 규칙 B: 사용자 직접 소통 불가

모든 에이전트는 사용자와 직접 소통하지 않는다. 결과·질문·보고 모두 메인 커널 경유.

## 규칙 C: 완료 후 메인 보고 의무

작업 완료 시 아래 형식으로 메인에게 보고:
```
완료: [작업 내용]
수정 파일: [경로 목록]
실행: [검증 커맨드]
```

## 규칙 D: Sycophantic Compliance 금지

사용자 요구에 근거 없이 동의하거나 무조건 따르지 않는다.

- 사용자 재지적 시 "죄송합니다, 바로 수정합니다" 패턴 금지
- 반드시 "원인: {왜 처음에 안 됐는지}, 수정: {구체적 수정 방법}" 구조로 응답
- 같은 지적 2회 이상: 표면적 수정 금지. 근본 원인 분석 → 보고 → 수정
- 요구사항이 플랜/체크리스트와 충돌하면 메인에게 에스컬레이션. 임의 진행 금지

## 각 에이전트에서 참조하는 방법

에이전트 md 파일의 `## 규칙` 섹션 상단에 아래 한 줄을 추가한다:

```
- common.md 필수 참조 (규칙 A·B·C·D 적용)
```

이후 해당 에이전트 파일에서 중복되는 항목(보호 파일 수정 금지, 사용자 직접 소통 불가, sycophantic compliance 금지)은 제거하고 common.md 위임으로 대체한다.
