# 플랜 작성 규칙

plan/context/checklist 작성 시 `.manuals/process/plan.md`를 읽고 따를 것.

## 핵심 (목차만)
- .plans/YYYYMMDD-작업명/ 폴더에 01_plan.md, 02_context.md, 03_checklist.md 생성
- 코드 작성 전 반드시 계획 먼저
- 세션 복구용 상태 스냅샷 포함
- 코드 변경 항목에는 대응하는 테스트 항목을 반드시 포함 (문서만 수정하는 항목은 제외)
- 상세 포맷/규칙은 `.manuals/process/plan.md` 참고
