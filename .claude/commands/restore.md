# /restore — 세션 복구

이전 세션에서 하던 작업을 이어받습니다.

## 프로세스

1. `.plans/` 디렉토리에서 가장 최근 폴더를 찾습니다 (`ls -t` 실제 수정 시간순, 폴더명 정렬 금지).
2. `.hook-state` 파일이 있으면 읽어 이전 상태 복구합니다.
3. 해당 폴더의 3대 문서를 읽습니다:
   - `01_plan.md` → 전체 목표 파악
   - `02_context.md` → 기술적 맥락 + 마지막 세션 상태 스냅샷 확인
   - `03_checklist.md` → 진행 상황 파악 (마지막 `[x]` 다음의 `[ ]`가 현재 할 일)
3. 사용자에게 현황을 **간결하게** 보고합니다:

```
📋 작업 복구: {작업명}
━━━━━━━━━━━━━━━━━━
🎯 목표: {01_plan.md 요약 1줄}
✅ 완료: {n}개 항목
⬜ 남음: {m}개 항목
▶️ 다음: {다음 체크리스트 항목}
━━━━━━━━━━━━━━━━━━
계속 진행할까요?
```

4. 사용자가 OK하면 다음 체크리스트 항목부터 실행합니다.

## `$ARGUMENTS` 사용
- 비어있으면: 가장 최근 `.plans/` 폴더
- 작업명이 있으면: `.plans/` 에서 해당 이름 검색

## 주의
- `.plans/`가 비어있으면 "진행 중인 작업이 없습니다. `/task-plan`으로 새 작업을 시작하세요." 안내
- 복구 시 관련 `.claude/skills/` 문서도 함께 참조
